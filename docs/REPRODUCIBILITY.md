# Reproducibility

Every numeric value reported in the paper is regenerated by this codebase. No value is hand-entered.

## The single command

```
cargo run -p cli --release -- reproduce
```

This:

1. Regenerates every deterministic vector under `/vectors`.
2. Runs the synthetic-population benchmark in `/bench`.
3. Diffs each regenerated output against the committed expected output.
4. Exits non-zero on any mismatch.

CI runs this command on every push (`.github/workflows/ci.yml`).

## What is bit-deterministic across hosts

The following are bit-identical on any host running the pinned toolchain:

- All cryptographic vectors in `/vectors` — Pedersen commitments under fixed blinding-factor seeds, Σ-protocol transcripts under fixed Fiat–Shamir nonces, Bulletproofs under fixed challenges, Merkle roots over fixed leaf sets, BSV txid/header reconstruction over fixed transaction bytes.
- Synthetic-population counts: invoice/payment counts, note sizes, batch counts, injected-fault counts, detection counts, clean-recalculable share.

The deterministic seed for the synthetic-population study is recorded in `docs/DECISIONS.md` D-008 (pending — populated at step 8).

## What is host-dependent

Only the wall-clock measurements in `/bench`. The benchmark harness reports median and IQR over many iterations. Hardware is recorded in `ENVIRONMENT` once a benchmark is captured.

## Self-test guarantees

The reproduction harness asserts:

1. Every committed vector is byte-identical to its regenerated counterpart.
2. Every benchmark median is within a stated tolerance band of the recorded reference run (drift outside the band fails the build and prompts a rerun with hardware recorded).
3. Every negative test that must fail does fail (e.g. collusive-false-origin commitment is not detected).
4. Every adversarial test (wrong path, wrong index, wrong root, missing/altered/duplicated shard) rejects.

## Live BSV verification (optional)

Default tests use offline fixtures committed to `/vectors/bsv/`. An optional path verifies against a live BSV header source — see the `--live` flag on `vaa verify`. This path is not in CI.
