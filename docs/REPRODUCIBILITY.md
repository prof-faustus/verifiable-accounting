# Reproducibility

Every reported value is regenerated by this codebase. No value is hand-entered.

## The single command

```
cargo run -p vaa-cli --release -- reproduce
```

This:

1. Regenerates every deterministic vector under `/vectors`.
2. Diffs each regenerated output against the committed expected output.
3. Exits non-zero on any mismatch.

CI runs this command on every push (`.github/workflows/ci.yml`).

## What is bit-deterministic across hosts

Under the pinned toolchain, every committed vector under `/vectors/` is bit-identical on any host:

- `vectors/merkle/genesis_v1.json` — BSV genesis block single-leaf root.
- `vectors/merkle/bsv_block_v1.json` — a real BSV mainnet multi-transaction block: txids in block order and the published `merkleroot`. Reconstructed leaf-to-root bit-for-bit by `cargo test -p vaa-merkle bsv_mainnet_block_merkle_root`.
- `vectors/study/storage_<N>.json` — Layer B storage / retrieval efficiency at population size `N`. Structural fields are bit-deterministic; wall-clock fields are reported separately and omitted from the committed vector.
- `vectors/study/simstudy_<M>.json` — Layer A inclusion + Layer B selective disclosure + fault detection counts at population size `M`. Bit-deterministic counts; honest record of the origin-falsehood boundary as **not detected** (by design).

Deterministic seeds:

- `vaa-simstore`: `SEED = 2_026_053_001` (defined at the top of `crates/simstore/src/main.rs`).
- `vaa-simstudy`: `SEED = 2_026_053_002` (defined at the top of `crates/simstudy/src/main.rs`).

## What is host-dependent

Wall-clock timings in the benchmark harness and in the simulation studies' phase timings. The harness reports median and inter-quartile range over many iterations; hardware is recorded in `ENVIRONMENT` once a benchmark is captured. No throughput figure is extrapolated.

## Fast CI points vs. full report points

CI runs the small/fast parameter points:

- `vaa-simstore --n 1024 --q 64 --vector-out vectors/study/storage_1024.json`
- `vaa-simstudy -m 200 --inclusion-sample 64 --vector-out vectors/study/simstudy_200.json`

Both write small JSON vectors that `vaa reproduce` diffs on every push.

The full report points (large `N`, large `M`) are run on the reporting machine and are not in CI:

- `vaa-simstore --n 262144 --q 1000` (or similar)
- `vaa-simstudy -m 100000 --inclusion-sample 1000`

Capture the host of the run in `ENVIRONMENT` alongside the figures.

## Self-test guarantees

`vaa selftest` asserts at runtime:

1. `bsv` double-SHA256 matches the known empty-input vector.
2. `merkle` reconstructs a synthetic 8-leaf tree and rejects a tampered leaf.
3. `accounting` checks the invoice-total identity and the AR roll-forward identity over disclosed records.
4. `proofstore` anchors + queries + verifies in both adversarial and proof-assistance modes.

`vaa reproduce` additionally asserts that every committed vector regenerates bit-for-bit.

## Live BSV verification (optional)

Default tests use offline vectors. An optional path verifies against a live BSV header source (your own BSV node or a public BSV explorer); not enabled in CI.
