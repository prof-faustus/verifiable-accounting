# Realistic quarterly-close example

A complete, end-to-end `vaa` workflow that mirrors a mid-sized firm closing its books for one quarter and anchoring each accounting record on BSV. The example is regenerated deterministically by `generate.py`.

## The hypothetical entity

**Aurora Mid-Cap Services Ltd.** — a fictitious mid-market services firm. All amounts are expressed in **minor units** (the smallest accounting unit chosen by the entity) so the figures fit in `u64` and the data round-trips through every layer cleanly.

For the period **2026-04-01 to 2026-06-30** (Q2-2026):

- 48 invoices issued to 12 distinct counterparties
- 43 invoices settled within the quarter
- 5 outstanding at quarter-end (rolled into closing AR)
- 1 credit note issued
- 1 receivable written off

## Files

| File | What it contains |
|---|---|
| `records.json` | All 93 records (48 invoices + 43 receipts + 1 credit + 1 write-off). Each record's leaf hash is `double_sha256(record_bytes)`. Suitable as input to `vaa prove`. |
| `ar_rollforward.json` | The Q2 AR roll-forward period totals, the identity check, and the equation. Verified by direct recomputation over the disclosed records. |
| `bundle_record_017.json` | A worked proof bundle for record #17, regenerated by `generate.py` via `vaa prove`. Verifiable via `vaa verify`. |

## How to use

```sh
# 1. Anchor the record batch (build the BSV-canonical Merkle root over the leaves).
vaa anchor --leaves <leaves.json>

# 2. Regenerate the proof bundle for record #17 from scratch.
vaa prove --records examples/realistic_quarter/records.json --index 17 \
          --out /tmp/bundle.json

# 3. Verify the pre-generated bundle (no live BSV node required).
vaa verify --bundle examples/realistic_quarter/bundle_record_017.json
```

The accounting identity check in `ar_rollforward.json` is plain recomputation:

```
AR_close = AR_open + Invoices − Receipts − CreditNotes − WriteOffs
```

The library `vaa-accounting` checks this directly against the disclosed records by recomputation; privacy comes from selective disclosure of only the queried records.

## Numbers (minor units)

| Line | Minor units |
|---|---|
| AR opening (2026-04-01) | 18,200,000 |
| Invoices issued in Q2 | 183,944,000 |
| Receipts in Q2 | 173,419,000 |
| Credit notes in Q2 | 1,800,000 |
| Write-offs in Q2 | 950,000 |
| **AR closing (2026-06-30)** | **25,975,000** |

Identity check: `18,200,000 + 183,944,000 − 173,419,000 − 1,800,000 − 950,000 = 25,975,000` ✓

(Exact figures are generated deterministically by `generate.py` from a fixed seed.)

## Reproducibility

Run `python generate.py` to regenerate every file in this directory from the seed. Requires the `vaa` binary on `PATH` (or `$VAA`).
