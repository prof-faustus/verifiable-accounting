# Contributing

This repository is the reference implementation for an academic paper. Contributions are welcome once the paper is publicly available; until then the repository is private and changes are limited to the author.

## Building

Requires the toolchain pinned by `rust-toolchain.toml` (Rust 1.85.0 stable, with `rustfmt` and `clippy`). On first checkout:

```
cargo build --workspace --release
cargo test  --workspace
```

## Style and lint

```
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
```

CI fails on any `fmt`, `clippy`, or test regression.

## Commit conventions

- One logical change per commit.
- Imperative present-tense subject ("Add Merkle proof verifier" not "Added"/"Adds").
- Body explains *why*, not *what* the diff already shows.
- No secrets, keys, or credentials in commits (see `SECURITY.md`).

## Tests

Every public function carries unit tests. The Merkle layer additionally carries property-based tests (`proptest`) and adversarial tests (wrong index, wrong root, wrong path, tampered leaf). Negative tests that *must* fail (e.g. the collusive-false-origin commitment boundary) are asserted in code, not papered over.

Run only the property suite:

```
cargo test -p merkle --features proptest
```

## Reproducibility

All numbers cited by the paper are regenerated by `cargo run -p cli --release -- reproduce`. CI runs this on every push; any drift fails the build. If you change a benchmark or a vector, regenerate the expected output and commit it in the same PR.

## Security

See `SECURITY.md` for vulnerability disclosure. Do not file public issues for security bugs.
